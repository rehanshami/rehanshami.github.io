The project

The device help content migration consisted of relocating approximately 2000 help articles. I completed this project from inception to final output through research, designing page templates, SEO optimization, and rewriting content in Spark branding and tone.

Duration: 1 month

Methods: Scoping content, wireframes, image editing, researching, SEO, and content rewriting.

Tools: Adobe AEM, Adobe Photoshop, and Figma.

Challenges, insights, and outcomes

The main challenge during this project was the timeframe. I designed a simple and intuitive design that we could use as a temporary solution. I researched and found all the relevant help articles on the device manufacturer's websites. While I worked on recreating all the content in-house, we were able to redirect customers to these relevant help articles.

The redirection approach was used for the mobile help content while all the modem help content was recreated in-house. This is because as Spark firmware is preloaded on all supplied modems, no relevant help articles could be found.

The help content is live. All modem related content has been recreated in-house, while mobile support content directs customers to external vendor support articles.
